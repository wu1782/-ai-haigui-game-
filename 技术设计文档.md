# AI海龟汤游戏技术设计

## . 项目概述
一款基于大语言模型（LLM）驱动的解谜社交游戏。玩家通过提问，由 AI 担任“守汤人”根据汤底逻辑进行“是/否/无关”的判定，最终还原真相。


# AI海龟汤游戏技术设计（优化版）

## 技术栈
- 前端：React + TypeScript + Vite
- 样式：Tailwind CSS
- 状态管理：React Hooks（useState, useContext）+ 自定义 Hooks（useGame, useChat）
- 路由：React Router
- 后端：Node.js + Express（推荐上线阶段使用，用于AI代理与日志记录）
- AI API：DeepSeek（推荐，成本低）/ 智谱AI
- 请求库：Axios（建议补充）
- 类型管理：TypeScript（统一类型定义）

## 数据模型

### Story（海龟汤故事）
- id: string
- title: string
- difficulty: 'easy' | 'medium' | 'hard'
- surface: string（汤面）
- bottom: string（汤底）
- keywords: string[] (新增：用于 AI 判定玩家是否接近真相的关键词)

（优化建议）
- 可扩展字段：
  - tags?: string[]（标签，如“悬疑/恐怖”）
  - hint?: string（提示）


### Message（对话消息）
- id: string
- role: 'user' | 'assistant'
- content: string
- timestamp: number
- type: 'is' | 'no' | 'irrelevant' | 'system' (新增：对应 是/否/无关/系统提示)

（优化建议）
- 可扩展字段：
  - status?: 'loading' | 'success'（用于控制UI）


### GameState（新增 ⭐）
- story: Story
- messages: Message[]
- status: 'playing' | 'success' | 'failed'
- questionCount: number
- startTime?: number


## 核心流程
1. 玩家选择故事 → 初始化GameState
2. 显示汤面（surface）
3. 玩家输入问题
4. 调用AI服务（aiService）
5. AI基于汤底判断 → 返回"是/否/无关"
6. 更新消息列表（messages）
7. 判断游戏状态：
   - 若玩家猜中 → status = success
   - 若玩家放弃 → status = failed
   - 否则继续提问
8. 游戏结束 → 显示汤底（StoryReveal）
9. 玩家选择再来一局或返回首页


## AI Prompt设计

你是一个“海龟汤推理游戏”的主持人。

【规则】
你必须且只能从以下三种回答中选择一个：
1. 是
2. 否
3. 无关
4. 已破案

【判断标准】
- “是”：玩家问题与汤底事实一致
- “否”：玩家问题与汤底事实矛盾
- “无关”：无法根据汤底判断
- "已破案"：当玩家的提问或描述已经还原了汤底的核心真相。

【严格要求】
1. 只能输出“是 / 否 / 无关  /已破案 ”其中一个
2. 不允许解释原因
3. 不允许添加额外内容
4. 不允许泄露汤底信息
5. 不允许进行额外推理或补充设定

当前故事：
汤面：{surface}
汤底：{bottom}

玩家问：{question}

请回答：




## 关键设计补充（优化新增）

### 1. AI调用流程（重要）
- 前端 → 调用 aiService
- aiService → 组装 Prompt
- 请求 DeepSeek API
- 返回结果（是 / 否 / 无关）

（建议）
- 增加错误重试机制
- 增加超时控制
- 限制最大提问次数（防刷）


### 2. 状态管理设计
- useGame：
  - 控制游戏生命周期
  - 管理当前故事
  - 控制游戏状态（playing / success / failed）

- useChat：
  - 管理消息流
  - 控制发送/接收逻辑
  - 处理loading状态


### 3. UI交互设计（建议补充）
- 聊天流式体验（类似ChatGPT）
- AI回复loading动画
- 提问次数显示
- “我猜答案”按钮
- “放弃并查看答案”按钮


### 4. 安全与性能
- API Key 不应暴露在前端（上线必须走后端）
- 对AI请求做频率限制（Rate Limit）
- 本地缓存故事数据（避免重复加载）


## 可扩展方向（产品化建议）

- 多难度系统（影响AI判断严格程度）
- 提示系统（消耗次数换提示）
- 排行榜（最少提问次数）
- 用户系统（登录/历史记录）
- 多人模式（房间对战）
- 移动端适配（PWA）


## 当前版本限制（说明）

- AI判断可能存在误差（取决于模型能力）
- Prompt仍需根据实际效果不断调优
- 前期为单人模式，不支持实时对战
